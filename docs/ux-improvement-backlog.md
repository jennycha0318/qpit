> 출처: UX ralph loop v2 — 8화면(레이아웃·탭바·로고 / 랜딩 / 인증 / 진단 / 결과 / 히스토리 / 프로필 / 디자인시스템) × 8 UX 차원, 화면별 Explore 에이전트 감사.

## ✅ 이번 라운드 적용
- ★ **결과 ▲▼ 델타 판정감 완화** — '이렇게 판단했어요' 아래 "점수 계산에 반영된 요인이에요. 당신이 부족하다는 뜻이 아니에요." 캡션 추가.
- **성향·궁합 카드 신뢰도 명확화** — 배지 '참고' → '점수 미반영'.
- **진단 진행바 보정** — `(qIndex/total)` → `((qIndex+1)/total)` (첫 질문부터 채워지고 마지막에 100% 도달).
- **btn-ghost 대비 개선** — 배경 0.45 → 0.68(글래스 위 가독성/WCAG).
- **로고 접근성** — 워드마크와 함께 쓰일 때 로고 `aria-hidden`(스크린리더 'Pacemaker' 중복 읽기 제거).
- **히스토리 상세 맥락** — Report 위에 '단계 · 일시' 메타 추가 + 뒤로가기 버튼화.
- **인증 접근성** — 로그인·회원가입 입력에 `id/htmlFor` 연결 + `autoComplete`(이메일/현재·새 비밀번호/이름) → 비밀번호 매니저·라벨 탭 지원.

## ✅ P1 — 전부 적용 (이번 라운드)
- ★ **결과 섹션 위계/그룹핑** ✅ — 3개 `<section>` 그룹(결과 / 다음 행동 / 근거·참고), 그룹 내 gap 좁게·그룹 간 mt-7, 각 그룹 소제목.
- ★ **위기 상담 카드 가시성** ✅ — needsSupport 시 CrisisResources를 최상단(점수 카드 앞)으로 이동(sticky 미사용, border-2 유지).
- **인증 실시간 검증 + 비밀번호 표시 토글** ✅ — 공용 `PwToggle`(눈 아이콘) + 이메일/비번/일치 onChange 힌트. reset/update도 id·autoComplete 보강.
- **진단 통합 진행 인디케이터** ✅ — `StepIndicator`(내 정보/상황/상대/설문 4단계, 현재 강조·지난 done), 설문 내 질문 진행바 병행.
- **자동진행 오선택 회복** ✅ — 선택 옵션 체크(✓)+'선택됨'+ring, '← 이전' 버튼형 승격으로 직전 수정 용이.
- **히스토리 삭제 기능** ✅ — `DeleteDiagnosisButton`(확인 모달) 목록/상세에 추가(RLS delete 활용).
- **디자인시스템 텍스트 크기 일관화** ✅(부분) — 캡션 11/11.5/12px → **12.5px 통일**(손글씨 가독성·일관성). ⬜ 하드코딩 hex→토큰, 헤더 스케일 체계화, 간격/라운드 정규화는 P2(리팩토링)로 잔류.

## ✅ P2 — 전부 적용 (이번 라운드)
- **결과 공유** ✅ — `ShareButton`(navigator.share, 미지원 시 클립보드 복사). 진단 결과·히스토리 상세 양쪽. _(이미지/PDF 내보내기는 로드맵)_
- **결과 risks 과다 시 접기** ✅ — 3개 초과 시 네이티브 `<details>`("N개 더 보기")로 접음.
- **hold vs msg 시각 구분** ✅ — hold=일시정지 아이콘+경고 톤, msg=종이비행기 아이콘+부드러운 톤.
- **상대정보 단계 가치 설명** ✅ — `<details>` "입력하면 뭐가 달라지나요?"(궁합·소통팁·나이차, 점수 미반영 명시).
- **게스트 결과 보존** ✅ — 게스트 결과를 localStorage 저장 → 가입 후 /diagnose 진입 시 자동 저장·복원.
- **히스토리 정렬/필터 + 날짜 그룹핑** ✅ — `HistoryList`(단계 필터 칩 + 오늘/이번 주/이번 달/그 이전 그룹).
- **프로필 저장 토스트 + 미저장 이탈 경고** ✅ — 성공 배너(2.5초) + dirty 시 `beforeunload`.
- **하드코딩 색 정리** ✅(부분) — 점수/톤 색·배지를 `lib/diagnose/colors.ts`로 공용화(Report/HistoryList/ShareButton 중복 제거). ⬜ 소프트 배경 hex의 Tailwind 토큰 클래스 전환은 P3 잔류.
- **파스텔 위 흰 글씨 대비** ✅(완화) — .btn-primary 흰 글씨에 미세 text-shadow로 가독성 보강. ⚠️ 완전 WCAG-AA 준수(그라데이션 darken)는 **브랜드/취향 결정** 필요(현 소프트 파스텔 유지 중).
- **헤더 스케일** — 변경 안 함(의도된 위계: 히어로 34 > 인증 30 > 페이지 26 > 질문 25, '글자 키우기' 요청과 정합).
- **인증 reset/update 접근성** ✅(P1에서 처리) — id/htmlFor·autoComplete 완료. (에러 메시지 레이아웃 점프는 경미 — 백로그)

## P3 — 적용/보류 (이번 라운드)
- ✅ **랜딩 폴리시** — 첫 카드 강조(border-2 primary), 버튼 hover(밝기/배경), pt 중복 제거, h1 leading 1.45.
- ✅ **입력 포커스 표기** — `.field-input:focus` 포커스 링(셀렉트 포함). ⬜ 네이티브 `<select>` 풀 커스텀 콤보박스는 보류(모바일 네이티브 픽커가 접근성상 더 나음).
- ✅ **손글씨 작은 글씨 가독성** — body `letter-spacing: 0.01em` + 캡션 12.5px 통일(P1).
- ✅ **빈 상태 온기** — 히스토리 첫 진단 카피에 격려·안전 보관 메시지.
- ✅(부분) **프로필** — 이름·이메일·로그인 방식 '수정 불가' 안내 추가. ⬜ 계정/데이터 삭제·탈퇴는 법적 보관·auth 사용자 삭제(서버 함수) 필요 → **founder 결정**(backlog).
- ⬜ **간격·라운드 정규화** — 시각 변화 위험 대비 효용 낮아 보류(현 값 일관성 충분). 디자인 토큰 정식화 시 함께.
- ⬜ **소프트 배경 hex → Tailwind 토큰 클래스** — 시각 중립 리팩토링, 후순위(점수/톤 색은 `colors.ts`로 공용화 완료).
- ⬜ **TabBar backdrop-blur 대체** — 글래스 아이덴티티 유지 우선(저사양 대체는 필요 시).
