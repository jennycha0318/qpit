# AI 컨설팅 로직 상세 설계

> PRD §6의 확장 문서. 이 문서가 차별화의 핵심(`web/src/lib/frameworks/`, `web/src/lib/ai/`)을 규정한다.
> 상태: 🚧 초안 — 채워나갈 것

## 1. 파이프라인 개요
```
사용자 입력(설문)
  → [정규화] 입력을 표준 변수로 변환
  → [프레임워크 규칙 레이어] 심리 이론 기반 가중치/제약 적용
  → [LLM 프롬프트] 구조화 프롬프트로 진단 생성 (JSON 출력 강제)
  → [후처리/검증] 가드레일·스키마 검증
  → 진단 리포트 렌더링
```

## 2. 입력 변수 (정규화 대상)
- 상황: 관계단계, 경과시간, 연락추세, 마지막 상호작용 톤
- 성향: 애착유형(본인/상대), 연애가치관, 갈등대응
- 맥락: 이별사유, 미해결갈등, 외부요인

## 3. 프레임워크 규칙 레이어 (핵심 차별점)
범용 LLM의 아첨(sycophancy)을 억제하고 일관된 전문가 판단을 재현하는 규칙.
- 애착이론: 상대 유형별 적정 접근 속도
- 재접촉 곡선: 이별 후 경과시간 ↔ 수용도
- 상호성·희소성: 과잉 연락 vs 적정 거리
- 손실회피: 무행동의 기회비용 제시
