## MVP 핵심 플로우 (현재 구현, `web/`)
```
랜딩 (/)  ← 로그인 상태면 /diagnose 로 자동 이동
 → 나이대 선택 (10대 / 20대 / 30대 / 40대 이상)  ← 10대면 '청소년 모드'
 → 고민 유형 선택 (썸 / 연애 중 / 이별 후)
 → 상황·성향 설문 (선택 시 약 320ms 자동 진행, 마지막 문항은 자유서술-선택)
 → [규칙 기반 진단 엔진 계산]  ← 결정적 규칙 (LLM/AI API 호출 아님)
 → 진단 리포트 (점수·타이밍/플랜·판단 근거·액션·리스크)
 → 추천 메시지 (또는 보류 안내)
 → CTA: 다시 진단하기(재진단) / 로그인 시 결과 자동 저장 · 게스트면 로그인 유도 카드
```
- **게스트(비로그인)도 진단 가능** — 진단·결과 전부 제공, DB 저장은 안 함.
- **로그인 시** 결과를 `diagnoses` 테이블에 자동 저장(saving→saved).
- 진단 엔진은 규칙 기반(결정적): `web/src/lib/diagnose/survey.ts`, `web/src/lib/diagnose/engine.ts`. LLM 기반 AI 개인화는 (계획) 단계.
- 자유서술 입력은 현재 진단 점수에 반영되지 않고 기록만 저장됨 (추후 AI 개인화에 활용 예정 · 미구현).
