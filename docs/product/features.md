## MVP (v1) — 고백/재회 타이밍 진단 (구현됨)

현재 `web/` MVP는 **규칙 기반(결정적) 진단 엔진**으로 동작합니다. LLM/AI API 호출이 아니라, 설문 응답에 가중치를 적용해 점수·플랜·근거를 산출합니다. (엔진: `../../web/src/lib/diagnose/engine.ts`, 설문: `../../web/src/lib/diagnose/survey.ts`)

- [x] 상황 입력 설문 (관계단계 분기: 썸 `crush` / 연애 중 `dating` / 이별 후 `breakup`)
- [x] 진단 리포트 (타이밍 점수 3~97 + 판단 근거 factors + 액션 + 리스크 + 연락 플랜)
  - ⚠️ 표기 주의: 현재 진단 로직은 **규칙 기반**입니다. "AI 진단"이라는 표현을 쓸 경우 LLM 개인화는 **계획** 단계임을 함께 명시하세요.
- [x] 추천 메시지 (단계·점수별 예시 메시지 / 보류 안내 `hold`)
  - 참고: LLM이 매번 새로 생성하는 "생성기"가 아니라, 규칙으로 선택되는 **예시 메시지**입니다. 동적 생성은 (계획).
- [x] 안전장치: 위기 상담 연결(needsSupport), 청소년(미성년) 모드(minor), 법적·윤리 고지 (`../../web/src/components/SupportNotices.tsx`)
- [x] 결과 저장/이력: 로그인 시 `diagnoses` 테이블에 저장, 히스토리 조회 (게스트도 진단 자체는 가능)

## 계획 / 미구현 (프로토타입·로드맵 단계)

아래 기능은 기획·프로토타입 단계로, 현재 MVP 코드에는 **포함되어 있지 않습니다.**

- [ ] 상대 메시지 해석 (카카오톡 대화 이미지 업로드 → 분석) — (미구현)
- [ ] LLM 기반 개인화 (자유 서술 `freeText`를 점수/리포트에 반영) — (계획)
  - 현재 자유 서술은 기록으로만 저장되며 점수에 반영되지 않습니다.
- [ ] 결과 공유 이미지 (리포트 이미지로 내보내기) — (미구현)
- [ ] 결과 HTML 다운로드 — (미구현)
- [ ] 결과 피드백 / 데이터 플라이휠 (`outcomes` 등 확장 테이블) — (계획)
- [ ] 집착 위험군 쿨다운 (재진단 제한 등) — (계획)
