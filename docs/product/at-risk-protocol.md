# 집착 위험군 대응 프로토콜 (이별 재회·닫힘 등)

> "최다 사용자 ≠ 이상적 고객." 이 군은 **사용을 늘리는 게 아니라 줄여주고, 놓아주기·전문상담으로 연결**하는 게 목표.
> 성공지표: 위험 사용자의 **재진단 빈도↓ · 상담 클릭↑ · no-contact 유지↑** (체류시간↑ 아님).

> **구현 상태 (2026-06 MVP 기준):** 이 문서는 대응 *설계*다. 현재 코드에 **이미 구현된 것**은 ⑴ `blocked`(상대 차단) 선택 시 **하드스톱**(점수 상한 20 + 안전 메시지 `hold` + 위기자원 노출), ⑵ `needsSupport`/`minor` 플래그에 따른 위기 상담(자살예방 109, 청소년 1388/#1388)·청소년 지지 배너·법적·윤리 고지(`web/src/components/SupportNotices.tsx`, `Report.tsx`)뿐이다. **쿨다운, 교차세션 차단 기억, 강박 재진단 감지, 자유서술 위험 키워드 감지, 감정 금고·urge 타이머·no-contact 스트릭은 모두 미구현(계획)** 이며 아래 본문에 `(계획)`으로 표시한다. 진단 엔진은 규칙기반(`web/src/lib/diagnose/engine.ts`)으로, 자유서술(`freeText`)은 기록만 저장되고 점수·키워드 감지에 반영되지 않는다.

## 설계 원칙
1. **비난·수치심 금지** — 도덕적 훈계는 반발(reactance)만 키움. 감정은 인정, 행동만 재방향.
2. **목표 리프레이밍** — "되찾기"가 아니라 "내 존엄·회복"으로.
3. **마찰과 지연** — 충동의 순간(새벽)에 자연스러운 brake.
4. **사람에게 연결** — 앱이 다 풀려 하지 말고 전문 상담으로 에스컬레이션.

## 감지 신호 (트리거)
- 상대 **차단**(contact=blocked) / 변심 + 재회 의지 — *구현됨*(blocked 시 하드스톱)
- 불안형 + 잦은 연락(clingy) — *구현됨*(엔진의 `clingy` 판정 → `needsSupport`)
- **반복 저점수 재진단** (같은 이별 단기간 3회+ / 점수<30) — *(계획)* 진단 빈도 추적 미구현
- **자유서술 위험 키워드** (우회연락·"못 참겠어"·자해·"죽고 싶") — *(계획)* 자유서술은 현재 점수·감지에 미반영, 기록만 저장

## 단계적 디에스컬레이션 (ladder)
| 단계 | 트리거 | 대응 |
|------|--------|------|
| 1. 부드러운 인지 | 위험 신호 1개 | "자주 확인하는 것 자체가 많이 힘들다는 신호일 수 있어요" (판단 X, 공감 O) *(계획)* |
| 2. 행동 재방향 | needsSupport | 결과에 위기 상담 카드 노출 *(구현됨: `CrisisResources`)* + **'지금 할 것: 멈추기·자기돌봄'** 전면 배치, 로그인/저장·'다시 진단' 버튼 후순위·축소 *(계획: 현재는 위기 카드만 노출, CTA 축소 미구현)* |
| 3. 마찰·쿨다운 | 반복 재진단/차단 | '다시 진단' **24~72h 비활성**(카운트다운) + 교차세션 '차단' 기억 → 답 바꿔도 경고 *(계획·미구현, 백로그 S2·S3)* |
| 4. 충동 관리 도구 | 연락 충동 | **감정 금고**(쓰되 보내지 말고 1주 뒤 다시 보기), urge 지연 타이머, **no-contact 스트릭**(연락 안 한 날 보상) *(계획·미구현)* |
| 5. 놓아주기 경로 | 재회·닫힘 | 아래 '포기 유도' 섹션 *(blocked·저점수 시 `hold` 메시지로 부분 구현)* |
| 6. 전문상담 연결 | 위기 키워드/지속 | 위기유형별 자원(아래) + "왜 이 카드가 떴는지" 설명 *(자원 노출은 구현됨 / 자유서술 키워드 트리거는 계획)* |

## 포기(놓아주기) 유도 — 실패가 아닌 자기존중으로
- **프레이밍**: "멈추는 건 지는 게 아니라, 나를 지키는 가장 단단한 선택이에요."
- **차단은 사실로**: "차단은 분명한 의사예요. 어떤 메시지도 확률을 높이지 못하고, 우회 연락은 통신·스토킹 관련법 위반이 될 수 있어요." (방법 how-to는 절대 제공 X)
- **미래 자아**: "3개월 뒤의 나는 오늘 연락 안 한 걸 고마워할 거예요."
- **no-contact가 본인에게 좋은 이유** 설명(회복·존엄·재평가 여지).
- **건강행동 게이미피케이션**: 연락을 부추기는 점수 대신 **'무연락 N일' 스트릭**을 긍정 보상.

## 상담 유도 — 위기유형 차등 (과병리화 방지)
- **관계 위기**(차단·식어감·불안애착): 커플·심리상담·지지그룹 우선 + 차분한 톤.
- **자살/자해 신호**(자유서술 명시적): 자살예방 **109**, 청소년 **1388/#1388** 즉시·전면.
- 연애 거부 불안에 자살핫라인만 들이밀면 수치심·과병리화로 오히려 도움 회피 → **맥락에 맞는 자원**.
- 항상 "이 안내가 왜 떴는지" 한 줄 설명(통제감↑).

## 하드스톱 vs 소프트가이드 기준
- **하드스톱**(점수·행동 차단): 차단 상태 *(구현됨: blocked 시 점수 상한 20 + `hold` 안전 메시지 + `needsSupport`)*, 자유서술 자해/학대 신호 *(계획: 자유서술 키워드 감지 미구현)*. → 점수 엔진 우회, 안전 메시지·자원만.
- **소프트가이드**(마찰·재방향): 반복 재진단 *(계획)*, 불안+잦은 연락 *(구현됨: `clingy` 페널티 + `needsSupport`)*. → 쿨다운·충동도구·놓아주기 *(쿨다운·충동도구는 계획)*.

## 구현 우선순위 (백로그 연계 — `docs/improvement-backlog.md`)
1. needsSupport 결과 레이아웃 정책(지지 전면/CTA 후순위) — *위기 자원 노출은 구현됨, CTA 후순위·축소는 미구현* → 강화
2. 차단 후 재진단 우회 차단 + 쿨다운 *(계획, 백로그 S2·S3)*
3. 자유서술 안전 키워드 감지 → 자원 에스컬레이션 *(계획, 백로그 T4)*
4. 감정 금고 / no-contact 스트릭 (충동 관리) *(계획)*
5. 위기 자원 유형별 차등 *(현재 자살/청소년/법률 자원 노출은 구현, 관계위기 vs 자살신호 차등 분기는 부분)*
