# 백업 기능 (MVP 이후 후보)

> 지금 MVP에는 넣지 않되, 검증 후 추가할 기능을 적립하는 곳.
> 특히 **비사용자 세그먼트**(타이밍 도구로는 안 잡히는 사람들)를 위한 인접 기능을 모아둠.
> 핵심 제품(heavy user: 재회러·짝사랑진지·위기회복)을 먼저 깊게 만든 뒤 확장.

## A. 비사용자 → 인접 기능

### 1) 이별 회복·클로저 트랙 (대상: '정리·닫힘' 군)
- 왜: 이들은 '연락 타이밍'을 안 물음. 니즈는 **회복·애도·매듭짓기**라 현재 제품과 핀트가 다름 → 별도 트랙.
- 무엇:
  - **무연락(no-contact) 트래커** + 무연락 N일 스트릭(긍정 보상)
  - **회복 저널** — 감정 기록, 주간 회고
  - 단계별 회복 콘텐츠(애도 5단계, 자기돌봄 루틴)
  - "왜 헤어진 게 나을 수도 있는지" 재평가 워크시트
- 시너지: at-risk 군의 '놓아주기' 경로와 컴포넌트 공유([at-risk-protocol](product/at-risk-protocol.md)).

### 2) 관계 정기 점검 (대상: '안정·개선' 군)
- 왜: 관계가 멀쩡해 평소엔 안 씀 → **주기적 가벼운 훅**으로 재방문 유도.
- 무엇:
  - 월 1회 **관계 건강 체크업**(짧은 진단 + 추세)
  - 기념일·중요 시점 알림, 커플 대화 프롬프트
  - 둘이 함께하는 가벼운 진단(선택)

## B. 핵심 제품 확장 (heavy user 심화)
- **대화 캡처 분석**(온도·응답속도·주도성) — 프로토타입 기능 web 이식
- **타임라인 추세 시각화** + 요인 변화(델타) — 최다 요청
- **AI 개인화(Claude)** — 자유서술 실제 반영([ai-personalization](product/ai-personalization.md))
- **결과 피드백 루프 + 자가발전**([data-and-accuracy](product/data-and-accuracy.md))
- **프리미엄/결제 퍼널** — 지불의향 세그(재회러·진지 썸) 수익화

## C. 아이디어 스톡 (미정렬)
- What-if 민감도 시뮬레이터(1~2변수 토글)
- 의사결정 카드 / 실행 가이드 분리
- 결과 유효기간(Validity Window) + 트리거 기반 재진단
- 결혼 의향·연령 타임라인 경로 / 장거리 변수
- LGBTQ+ 공개단계 차원 / 성중립 템플릿
- 점수 모델 투명성 페이지

> 상세 개선 항목은 [improvement-backlog.md](improvement-backlog.md), 세그먼트 분석은 [customer-segmentation.md](customer-segmentation.md) 참조.
