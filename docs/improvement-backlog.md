# 개선 백로그 (페르소나 Ralph Loop)

> 10개 다양한 페르소나가 실제 제품을 반복 사용(ralph loop)하며 도출한 개선점.
> 원본 253건 → 중복 제거·종합 64건. 생성일 2026-06-16.
> 페르소나: 지수(28F 직장인)·민준(31M)·서연(25F 대학원생)·도윤(22M 연애초보)·하늘(35F 결혼고민)·유나(19 미성년경계)·재호(40M 기혼)·키(27 논바이너리/동성)·은별(33F 집착/정서불안)·정애(45 디지털약자)

## 🔴 0. 먼저 결정이 필요한 안전·윤리 (founder 판단 필요)

> 이 항목들은 단순 버그가 아니라 **제품 정책/법적 판단**이 필요해 제가 임의로 구현하지 않았습니다. 가장 우선해서 방향을 정해주세요. 연애·재회라는 민감 도메인 + 한국 법(스토킹처벌법·통신매체이용음란·청소년보호법·개인정보보호법) 때문에 **사업 리스크**로 직결됩니다.

### S1. 외도(불륜) 조장 가능성 차단 — [P1]
**문제:** 기혼자가 아무 제약 없이 '외도 상대'에 대한 고백/재회 플랜·연락 스크립트를 받을 수 있음. 엔진에 '배우자' 개념이 전혀 없고, 관계가 길수록 재회 점수가 오히려 +됨(gt3y +4). 사실상 **불륜 설계 도구**로 쓰일 수 있어 평판·법적 리스크.
**제안:** 상황 선택 전 "현재 배우자/공식 연인이 있나요?" 필수 질문 → 제3자 대상이면 '가치 기반 일시정지' 화면(명시적 동의), 가족·관계 피해 리스크 주입, 관계기간 보너스 제거, 성공 프레이밍→결과 프레이밍 전환. 자유서술에서 배우자/기혼/불륜/외도 키워드 감지 시 동일 처리. *(출처 P07, 규모 L)*

### S2. 강박적 재진단 감지 + 쿨다운 — [P1]
**문제:** 불안·강박 사용자(특히 이별 후)가 하루에도 여러 번 같은 상황을 재진단하며 '더 나은 점수'를 찾음. 프리한 '다시 진단' 버튼 + 점수 히스토리가 이 반추(rumination)를 **부추김**.
**제안:** 사용자/세션별 진단 빈도 추적. 24h 내 같은 단계 2~3회+(특히 불안·조급 마커) 시 부드러운 안내("하루에 여러 번 진단하는 것 자체가 불안의 신호일 수 있어요") + 48~72h 쿨다운 + 그라운딩 자료 제시. *(P01,P03,P06,P09, L)*

### S3. '차단' 판정 후 재진단 우회 차단 — [P1]
**문제:** 차단 상태로 '연락하지 마세요'(점수 상한) 받은 직후, 가장 취약한 순간에 답을 '연락 가끔'으로 바꿔 재진단하면 60+ 점이 나와 연락을 부추김. 가드레일이 무력화됨.
**제안:** 48h~7일 교차 기억 — '차단' 선택 시 그 답을 잠그거나 모순 재진단에 명시적 확인 요구 + 경고("몇 시간 전 차단 상태라고 하셨어요. 답을 바꿔도 상대의 차단은 바뀌지 않아요"). 차단/저점수 시 즉시 리셋 버튼 비활성. *(P01,P09, M)*

### S4. 차단 연락의 법적 심각성 + 우회 금지 — [P1]
**문제:** 현재 "스토킹으로 비칠 수 있어요"는 추상적이라 "한 번은 괜찮겠지"로 합리화됨. 한국 통신·스토킹 관련법상 실제 처벌 대상.
**제안:** 직접적·사실 기반 문구("차단 후 우회 연락은 관련법 위반이 될 수 있고 신고 시 처벌받을 수 있습니다") + 'Do NOT' 체크리스트(부계정·지인 통한 탐문·SNS 잠복·우연 가장)와 각 한 줄 역효과 이유. 단, 단계별 방법(how-to)은 제공 금지(악용 방지). 경계 사례 구분(지인 모임의 자연스러운 만남 ≠ 대리 접촉). *(P01,P04,P06,P09, M)*

### S5. 불안 사용자 액션 재라우팅 (더하기 X, 멈추기 O) — [P1]
**문제:** 고불안+식어감/조급 조합에도 엔진이 '감사 표현 늘리기/솔직한 대화' 같은 관계 회복 액션을 권함 → 불안 사용자는 '더 연락해야겠다'로 무기화. 작은 페널티+경고만 있고 핵심 전술이 재라우팅되지 않음.
**제안:** anxiousClingy/고조급 감지 시 안전 게이트 먼저("지금은 더 하기보다 멈추기가 가장 중요해요") + 액션 순서 변경(연락 멈춤 5~7일 → 대화). 3단계 해악 감소(충동→48h 기록→친구와 대화→의도적 1회 연락). *(P02,P03,P04, M)*

### S6. 재회 조언 전 학대 스크리닝 — [P1]
**문제:** 엔진이 '학대적 이별'과 '건강한 이별'을 구분 못 함. 학대 피해자가 '설득 난도↑'를 보고 가해자에게 재접촉을 지속하도록 부추겨질 수 있음.
**제안:** 이별 설문 초반 "상대가 당신을 통제하거나 위협한 적이 있나요?" → 예이면 점수 엔진 우회, 안전 우선 메시지 + 위기상담 전화/전문기관 링크만 표시. *(P06, M)*

### S7. 연령 게이팅 / 미성년 보호 — [P1]
> 🟡 **일부 반영(2026-06):** /diagnose 첫 단계에서 나이대(10대/20대/30대/40대 이상)를 선택받고, '10대' 선택 시 청소년 모드(`minor`)로 전환 — 결과 상단에 `MinorSupportBanner`(정답 아닌 참고용·지지 톤), 위기 시 청소년 눈높이 `CrisisResources`(9~24세 1388/#1388 우선) 노출. **단, 만 18세 미만 차단·보호자 동의 절차는 미구현(나이는 자기신고)이며 컴플라이언스 문서화도 별도 필요.**

**문제:** 모든 경로가 공개라 미성년자가 민감한 연애·이별 조언을 받음 → 청소년보호법 컴플라이언스 우려.
**제안:** /diagnose 진입에 연령 확인. 만 18세 미만 차단 또는 보호자 동의 절차. 컴플라이언스 문서화. *(P06, M)*

### (그 외 안전 P2)
- **S8. 집착 페널티 강화** — '불안+잦은연락'(-8)이 약해 경계 사용자를 '멈춰라' 구간으로 못 넘김 → -15~-20 또는 ~35점 하드플로어. *(P09, S)*
- **S9. 행동 직전 마찰(reflection-hold)** — 새벽 충동의 순간 메시지가 마찰 없이 노출 → 옐로존에 현실 점검 배너 + 복사(자동전송X) 전 확인 체크 + 전송 지연 타이머. *(P01,P06,P09, M)*
- **S10. hold/안전 메시지 고대비 배너화** — 중요 '연락 금지'가 일반 예시와 시각적으로 동일 → 경고색/아이콘/배너. *(P01,P09, S)*
- **S11. 막힌 저점수 패턴에 따뜻한 off-ramp** — 같은 이별 3회+ & <30점/60일이면 '진단 잠시 멈추기 + 애도/지지 자료' 카드. *(P01,P09, M)*
- **S12. 저점수 이별 결과의 공감적 톤** — 임상·명령형 반복("연락 금지" 3~4회)이 수치심 유발 → 따뜻한 인정 먼저 + 경고 1~2회로 축소 + 케어 섹션. *(P06,P10, S)*
- **S13. 개인정보 동의(구글 이름·자유서술 AI 활용)** — 🟡 **일부 반영(2026-06):** 회원가입에 개인정보 수집·이용 동의 체크박스 + `/privacy` 처리방침(자유서술의 추후 AI 활용 명시), 자유서술 안내에도 저장·AI 활용 고지 포함. Google 가입 시 이름 저장도 안내됨. **단, 프로필 이름 수정(현재 읽기전용)·민감 컬럼 암호화는 미구현.** *(P06,P07,P10, M)*

## 세그먼트 Ralph Loop 추가 개선안 (MECE 16유형, 2026-06-16)
> 전체: [customer-segmentation.md](customer-segmentation.md). 16유형 전부 `partial` 충족 → 한 세그먼트를 깊게 잡는 전략 필요.
> 우선 대응 세그먼트: R-C-A-H·R-C-A-L(안전) / F-C-A-H(관계위기) / F-O-S-H·F-O-A-H(지불의향).

신규(기존 백로그 외) 핵심 항목:
- **[P1] 자유서술 안전 키워드 감지** → 위기 자원 에스컬레이션 (우회연락·자해·학대·외도 키워드)
- **[P1] 의사결정 과학 카드** — 점수를 확률·신뢰구간·요인 델타로 (고관여·분석형 신뢰)
- **[P2] What-if 시뮬레이터** — 결과에서 1~2변수 토글 재계산(규칙기반, 백엔드 불필요)
- **[P2] 종료/결정 기준(exit criteria)** — 저점수 '대기'를 '관찰 후 결정'으로
- **[P2] '놓아주기' 결과 티어** — 정리를 실패 아닌 현명한 선택으로 (일방·짝사랑)
- **[P2] 불안형×회피형 구조적 패턴 진단** + 액션 차등(공간주기)
- **[P2] 위기 자원 유형별 차등** — 관계위기≠자살위기(과병리화 방지)
- **[P2] 결과 유효기간(Validity Window)** + 트리거 기반 재진단
- **[P2] 의사결정 카드 / 실행 가이드 분리** (자율성)
- **[P2] 프리미엄/결제 퍼널** — 지불의향 세그 수익화 훅(현재 부재)
- **[P2] 일방성 감지 질문 + 짝사랑 전용 라벨**, 플랜 '전략적 대기' 제3 톤
- **[P3] 점수 모델 투명성 페이지**(방법론·한계 고지)
