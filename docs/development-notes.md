## 1. 제품 개요
- **이름:** 큐핏 (영문 **Qpit**). 2026-06 "Pacemaker"에서 리브랜딩. **UI 노출 텍스트는 한국어 "큐핏"으로 통일.**
- **한 줄 정의:** AI 연애 **타이밍 의사결정 컨설팅** 웹앱 (썸/짝사랑/연애/이별 단계별로 "지금 어떻게·언제 행동할지" 분석).
- **포지셔닝:** "연애 컨설팅"(코치/코칭 아님). 페르소나 = "AI 연애 컨설턴트".
- **해자(moat):** 모델 자체가 아니라 그 위의 도메인 시스템(구조화 설문 + 심리 프레임워크(반-아부) + 타임라인 + 결과 피드백 + 신뢰 브랜드).

## 3. AI 아키텍처
- **결과 해석 라우트:** `POST /api/interpret` — `{stage, answers, minor, name}` 수신 → **서버에서 규칙 엔진 재계산**(점수·타이밍·행동·주의는 규칙이 결정) → **Claude가 '해석(interpretation)·문구(message)'만 생성**. 구조화 출력(json_schema).
- **모델 결정(2026-06):** **결과 해석 = `claude-opus-4-8`** (고품질). **챗봇(추후) = `claude-sonnet-4-6`.** (라우트 `MODEL` 상수.)
- **시스템 프롬프트 프레임워크:** 애착이론, 근거기반(반-아부), 안전 최우선, 미성년 배려, 집착/스토킹 금지. **규칙 구속 완화(2026-06-26):** 점수·타이밍·안전(보류·차단·위기·미성년) '방향'은 규칙 SoT로 잠그되, 해석·코칭·문구는 규칙의 정형 액션에 갇히지 말고 사용자·회고에 맞춰 자유롭게 발전(반복 피로 완화). 단 타이밍·안전 모순·집착·우회연락·거부 무시는 절대 금지(안전 carve-out, 차단 케이스 검증 OK). **호칭:** 로그인=`OO님`, 비로그인=`당신`. **반말("너·네가") 금지.**
- **폴백:** 학대 케이스/보류(hold)/키 미설정/오류/거부 → 규칙 결과로 자동 폴백 → **키 없이도 앱 동작.**
- **클라이언트 플로우:** 설문 완료 → **'분석 중' 화면** → AI 보강 완료(또는 25초 타임아웃 폴백) 후 결과 페이지 전환 → 저장 1회.

## 8-2. 결과 피드백 루프 & 백로그 (2026-06-25)
- ✅ **#1 개인 맞춤 (구현·검증)**: 재진단 시 직전 진단의 통찰 + 예측 + 사용자 피드백(예측 적중 맞음/반반/틀림, 그때 결과 잘됨/그저그럼/아쉬움 — `diagnosis_outcomes`)을 `interpret`에 전달 → "지난번에 통한 방법은 이어가고 안 통한 건 바꾸기"로 개인화. `diagnose/page.tsx` 마운트에서 outcome 조회→`prevInsight` 합성, interpret 프롬프트가 회고 반영. 테이블 없으면 통찰만(graceful).
- ✅ **학습 계측·개인화 확장 (2026-06-27, 암호화 없이 '쌓이는 출력'으로 학습 — 설계 워크플로 결론)**: ① `prevInsight`를 **직전 1건 → 최근 5건 이력 + 누적 예측 적중률**로 확장('나를 기억'→'나에 대해 학습', `diagnose/page.tsx`에서 `diagnosis_outcomes` 일괄 조회). ② **계측(소급 불가라 지금부터)**: `engine.ts`에 `factorSignature`(상황 패턴 해시 그룹핑)+`RULE_VERSION`, `interpret`에 `predictionType`(enum 8종)+`MODEL/PROMPT_VERSION` → `result` jsonb 저장. = "어떤 상황 패턴에 무슨 예측이 실제 맞았나" 집계·개선 전후 비교의 전제(검증 완료). 다음(백로그): 주간 집계 리포트 스크립트(#3), 필요 시 raw_inputs 암호화 테이블. **핵심 원칙: 학습=구조화 신호(factors+outcomes)로, 민감 원문은 학습 전제 아님.**
