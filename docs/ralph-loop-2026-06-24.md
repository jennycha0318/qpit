## 3. 가장 중요한 발견: "만족 7.3, 추천 −83"의 역설
만족도는 '괜찮다'(7점대)인데 **아무도 적극 추천하지 않는다**(추천 0). 점수가 6~7에 갇힌 이유는 일관됩니다 — **진단(머리)은 신뢰하지만, 상담 경험(마음)이 따뜻하지 않아 "남에게 권할 만큼은 아니다"**. 즉 큐핏은 지금 **정확한 분석가**이지만 사용자가 원한 **따뜻한 상담가**는 아닙니다. 이 간극이 NPS를 끌어내리는 단일 최대 원인입니다.

흥미로운 반례: **솔직·냉정함 자체는 강점**입니다(p03·p09 최고점 — "듣고 싶은 말 대신 필요한 말"). 문제는 냉정함이 아니라 **반복·피상·정서적 동반 부재**입니다.

## 4. 검증된 7개 테마 (적대적 3렌즈 검증 통과)
- **T1 [높음] 진단은 정확하나 상담이 피상적 — '분석가 vs 상담가' 괴리** (p01·p07·p10·p11·p12). 진단→곧장 처방으로 직행, 감정 명명·정상화 단계가 없음. → "머리는 납득, 마음은 위로 안 됨". *원인: interpret·chat 프롬프트가 '신호 분석+행동 처방'에만 최적화.*
- **T2 [중~높음] '앵무새 처방전' — 모든 답이 하나의 행동으로 수렴** (p01·p04·p08·p09). 서로 다른 질문인데 3턴 답변이 모두 '이번 주 만나자 보내기 / 연락 끊기'로 귀결 → 3번째쯤 신뢰 급락. *원인: chat이 질문 유형을 분기하지 않고 단일 처방을 재포장.*
- **T3 [높음] '그 다음엔?'이 없다 — 플랜B·사후 감정관리 부재** (p04·p05·p06·p09). 1차 행동만 주고 실패·무반응 시 분기가 없어 결정을 다시 사용자에게 떠넘긴 느낌 → 원점 회귀(다시 연락) 위험. *원인: 엔진 plan과 프롬프트가 단일 next-action에 머묾(특히 이별·짝사랑).* 
- **T4 [높음] 추천 문구(message) 결손 — 가장 취약한 순간에 빈 카드** (p02·p03·p06·p07·p09·p11). 차단 직후·장기 미련 등에서 `message`가 빈 문자열로 반환(`wantMessage=!d.hold` 설계). p07 "차단 직후 한 줄 위로 완전 부재 — 최악의 결함". *원인: hold 케이스의 빈 message 폴백 미정의 = 구조적 결함.*
- **T5 [중] 교과서·문어체로 인한 실행 장벽** (p11·p12). '첫째/둘째/셋째' 강의체라 특히 10대가 실제로 못 씀. *원인: 사용자 말투로 변환하는 보정 단계 부재, 카톡 톤 분석을 채팅에 미전달.*
- **T6 [중] 진단해놓고 처방 안 함 — 심리 라벨 후 전략 공백** (p03·p04·p05). '회피형 가능성' 정확히 짚고도 유형별 대응 없이 '기다려보라'로 종료 → 통찰의 절반만 전달. *원인: 애착 유형별 플레이북이 프롬프트에 지식으로 없음.*

### 가장 레버리지 큰 3가지(즉시 권장)
1. **P1 — 감정 명명·정상화 1문장**: 한두 문장 추가라는 최저비용으로 NPS의 1차 원인('위로 결핍')을 직접 공략. interpret의 interpretation 첫 문장에서 사용자가 freeText에 쓴 단어를 인용해 감정을 명명·정상화한 뒤 해석으로 진행. chat 구조도 `① 감정 인정 → ② 결론 → ③ 이유 → ④ 한 가지 행동`으로.
2. **P2 — 빈 message 제거**: message 점수가 1~3에 집중돼 분석 만족도를 직접 깎는 구조적 버그. 연락을 권하지 않는 상황엔 **상대가 아닌 '나에게 건네는 위로 한 줄'**을 항상 채워 빈 카드가 절대 안 뜨게.
3. **P3 — 앵무새 처방 차단**: 채팅을 (a)상대 마음 (b)내 행동·감정 (c)표현·문구 (d)실패 대처로 분기하고, 직전 답변에서 권한 핵심 행동을 같은 말로 반복 금지. 다회 상담의 신선도·체감 가치를 회복.
