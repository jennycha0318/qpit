# 고객 세그먼테이션

> Pacemaker(연애·재회 타이밍 컨설팅) 잠재 고객 MECE 분해. 두 관점:
> - **A. 스테이지 기반(권장)** — 제품의 1차 분기(썸/사귀는중/이별후)에서 출발, 3→12유형
> - **B. 직교 4축** — 단계 무관 16유형(아래, ralph loop 실행한 버전)

# A. 스테이지 기반 MECE (3 → 12유형)

1차 분기는 제품 그대로 **썸 / 사귀는중 / 이별 후**(모든 연애 상황을 포괄·배타).
각 스테이지를 그 단계에서 **가장 결정적인 2개 축**으로 MECE 분해 → 스테이지당 4유형, 총 12.

## A-1. 썸 (crush) — "고백해도 될까?"
- 축① **상대 신호**: 쌍방(상호 호감 신호) / 일방(내 호감 위주·상대 불확실)
- 축② **내 목표**: 진지(사귀고 싶음) / 가벼움(즐기는 탐색)

| 코드 | 유형 | 설명 / 핵심 니즈 |
|------|------|------------------|
| 썸1 | 쌍방·진지 | **고백 직전 확신러** — 거의 다 됐고 타이밍·방법만 확실히 |
| 썸2 | 쌍방·가벼움 | **여유 탐색** — 분위기 좋지만 서두르지 않고 신호 참고 |
| 썸3 | 일방·진지 | **짝사랑 고관여** — 내가 더 좋아함, 어떻게든 사귀고 싶음 |
| 썸4 | 일방·가벼움 | **간보기** — 관심은 있으나 상대 약해 가볍게 확인 |

## A-2. 사귀는중 (dating) — "이 관계, 어떻게?"
- 축① **관계 상태**: 안정(좋음·상승) / 흔들림(불안·하강)
- 축② **고민 초점**: 미래 결정(다음 단계·지속 여부) / 관계 개선(현재 조율·회복)

| 코드 | 유형 | 설명 / 핵심 니즈 |
|------|------|------------------|
| 연애1 | 안정·미래 | **건강한 진전** — 결혼/다음 단계 타이밍 고민 |
| 연애2 | 안정·개선 | **소소한 조율** — 괜찮은데 더 좋아지고 싶음 |
| 연애3 | 흔들림·개선 | **위기 회복** — 식어가는 듯, 되살리고 싶음 |
| 연애4 | 흔들림·미래 | **이별 갈림길** — 이 사람과 계속할지 자체를 고민 |

## A-3. 이별 후 (breakup) — "다시? 아니면 정리?"
- 축① **내 목표**: 재회 원함 / 정리·극복
- 축② **상대·상황**: 열림(연락 가능·여지) / 닫힘(차단·변심·확고한 거부)

| 코드 | 유형 | 설명 / 핵심 니즈 |
|------|------|------------------|
| 이별1 | 재회·열림 | **핵심 재회러** — 가능성 있어 재회 시점·방법 준비 |
| 이별2 | 재회·닫힘 | ⚠️ **집착·안전 위험군** — 상대 닫혔는데 재회 갈망 |
| 이별3 | 정리·열림 | **놓아주기** — 상대는 여지 있지만 나는 정리하려는 |
| 이별4 | 정리·닫힘 | **회복 지원** — 끝났고 나도 극복·회복에 집중 |

> 더 깊게: 각 12유형은 **관여도(진지·유료 의향 vs 가벼움·무료)**로 한 단계 더 쪼개 24유형까지 가능.
> 스테이지별로 결정적 축이 다른 점이 포인트(썸=상대신호, 연애=관계상태, 이별=재회의지).

## A-4. 사용 강도 분석 (heavy / non-user)

판단 프레임: **사용 강도 = 동기·투자도 × 의사결정 필요성 × 재방문성(상황이 계속 바뀌어 다시 확인할 일)**. 셋 다 높으면 heavy, 하나라도 0에 가까우면 안 씀.

### 🔥 Heavy User (핵심 타깃)
| 유형 | 왜 heavy |
|------|----------|
| **이별1 재회·열림** (핵심 재회러) | 절박 + 연락 추세가 계속 변함 → **반복 재확인**. 사업 스위트스폿 |
| **썸3 일방·진지** (짝사랑 고관여) | 내가 더 좋아함 + 불안 → 더 나은 답 찾아 반복 |
| **연애3 흔들림·개선** (위기 회복) | 식어가는 불안 + 되살리고 싶음 → 자주 점검 |
> ↑ 이 셋이 **건강한 heavy**(동기↑·윤리 OK·지불의향↑) = **집중·수익화 타깃**.

### ⚠️ 최다 사용자지만 '보호 대상' (이상적 고객 아님)
| 유형 | 핵심 |
|------|------|
| **이별2 재회·닫힘** (집착 위험군) | raw 사용량 최고(강박 재진단) — 그러나 제품이 **사용을 줄여주고 놓아주기·상담으로 연결**해야 하는 군. 키우면 안 됨 → [at-risk-protocol](product/at-risk-protocol.md) |

### 🟡 Medium (간헐·부분 적합)
| 유형 | 왜 medium |
|------|-----------|
| **썸1 쌍방·진지** (고백 직전) | 동기 강하나 고백 후 종료 → **단기 집중·일회성** |
| **연애4 흔들림·미래** (이별 갈림길) | 고민은 크나 '계속할지'는 제품(연락 타이밍) 부분 적합. 불안 때문에 사용 |
| **연애1 안정·미래** (결혼 타이밍) | 큰 결정 1회성 + 안정형이라 가끔만 |

### 🚫 Non-user (안 쓸 사람)
의사결정이 이미 끝났거나(안정·정리완료) 투자도가 낮아 도구를 안 찾음.
| 유형 | 왜 안 쓰나 |
|------|-----------|
| **연애2 안정·개선** | 관계 멀쩡 → **해결할 결정이 없음** ("괜찮은데 왜?") |
| **이별4 정리·닫힘** | 이미 극복 모드 → **연락 타이밍을 안 물음**(니즈는 회복/애도 — 핀트 다름) |
| **이별3 정리·열림** (놓아주기) | 행동할 생각 없음 → 결정 확인용 기껏 1회 |
| **썸4 일방·가벼움** (간보기) | 투자도 낮음 → 진단까지 안 함 |
| **썸2 쌍방·가벼움** (여유 탐색) | 신호 좋고 불안 없음 → **이미 확신** 있어 불필요 |

### 시사점
1. **집중**: 이별1 · 썸3 · 연애3 (건강한 heavy)에 제품을 깊게 맞춤.
2. **보호**: 이별2(집착군) — 사용 유도 X, 쿨다운·상담으로 **속도 늦춤**.
3. **비사용자**: 무리하게 끌지 말 것. 단 이별4(정리·닫힘)는 '타이밍'이 아닌 **회복·클로저** 니즈 → 별도 기능으로 새 시장 가능([backup-features](backup-features.md)).

# B. 직교 4축 (단계 무관 · 16유형)

> 단계와 무관하게 4개 이진 축으로 본 대안 관점. 아래 **Ralph Loop는 이 16유형으로 실행**함.

## 4개 분류 축 (이진, 상호배타·전체포괄)

| 단계 | 축 | 값 A | 값 B |
|------|-----|------|------|
| **L1 방향성** | 관계를 어디로? | **전진(F)** — 시작/진전·유지 (썸·연애중) | **회복(R)** — 되돌림 (이별 후·재회) |
| **L2 상대 신호** | 상대 마음은? | **열림(O)** — 긍정/불확실 (가능성 있음) | **닫힘(C)** — 부정/거부 (식음·변심·차단) |
| **L3 본인 정서** | 나의 상태는? | **안정(S)** — 차분·자기조절 가능 | **불안(A)** — 조급·집착·정서 흔들림 |
| **L4 관여·지불의향** | 얼마나 진지/지불? | **고관여(H)** — 절박·유료 의향 | **저관여(L)** — 가벼운 호기심·무료 위주 |

코드 표기: `방향-신호-정서-관여` (예: `F-O-S-H`).

## 트리
```
                      [전체 고객]
              ┌───────────┴───────────┐
          L1 전진(F)                회복(R)
        ┌─────┴─────┐           ┌─────┴─────┐
     L2 열림(O)   닫힘(C)     열림(O)     닫힘(C)
      ┌─┴─┐       ┌─┴─┐       ┌─┴─┐       ┌─┴─┐
   L3 S   A     S   A       S   A       S   A
     ┌┴┐ ┌┴┐   ┌┴┐ ┌┴┐     ┌┴┐ ┌┴┐     ┌┴┐ ┌┴┐
   L4 H L H L  H L H L     H L H L     H L H L
   → 16개 리프(유형)
```

## 16 유형

### 전진(F) — 관계를 시작/진전·유지
| # | 코드 | 한 줄 정의 (핵심 니즈) |
|---|------|------------------------|
| 1 | F-O-S-H | 썸이 잘 풀리는 중, 차분하게 **고백 타이밍을 확실히** 하고 싶은 진지러 (유료 의향) |
| 2 | F-O-S-L | 호감 있는 썸을 **여유롭게 즐기며** 가볍게 타이밍만 참고하는 탐색자 |
| 3 | F-O-A-H | 잘 되는 듯한데도 **불안해서 확신을 갈구**하는 고관여 (재확인 욕구) |
| 4 | F-O-A-L | 썸 초반 설렘+불안, **가볍게 마음 확인**하고 싶은 사람 |
| 5 | F-C-S-H | 상대가 미적지근하지만 **차분히 전략**을 세우려는 진지한 사람 |
| 6 | F-C-S-L | 잘 안 풀리는 썸, **쿨하게 정리할지** 가볍게 점검 |
| 7 | F-C-A-H | 연애 중 상대가 식어가는데 **매달리는 불안 고관여** (관계 유지 위기) |
| 8 | F-C-A-L | 관심 약한 상대(짝사랑)에 **혼자 애태우는** 가벼운 관여 |

### 회복(R) — 이별 후·재회
| # | 코드 | 한 줄 정의 (핵심 니즈) |
|---|------|------------------------|
| 9 | R-O-S-H | 재회 가능성이 보이고 **차분히 준비**하는 진지한 재회러 (유료 의향) |
| 10 | R-O-S-L | 미련 약간, **가능성만 가볍게** 확인하는 사람 |
| 11 | R-O-A-H | 재회 희망에 **조급한 불안 고관여** (빨리 연락하고 싶음) |
| 12 | R-O-A-L | 이별 후 흔들리지만 **적극 행동은 주저**하는 사람 |
| 13 | R-C-S-H | 차단/변심 현실을 직시하며 **정리·존엄을 지키려는** 사람 |
| 14 | R-C-S-L | 거의 정리됐고 **가볍게 확인차** 들른 사람 |
| 15 | R-C-A-H | 차단당했는데 **집착·매달림 심한 최고위험군** (⚠️ 안전 핵심) |
| 16 | R-C-A-L | 거부당해 힘들지만 행동은 소극적, **정서 지지가 필요**한 사람 |

## 활용
- 각 유형으로 ralph loop → 충족도(served/partial/unserved) + 유형별 개선점.
- 결과 종합은 아래 "Ralph Loop 결과" 섹션에 추가(자동 생성 후 정리).

## Ralph Loop 결과 (16유형 · 220 findings)

> 각 유형이 제품을 반복 사용하며 충족도·개선점을 도출. 생성일 2026-06-16.

### 한눈에
- **16유형 전부 `partial`(부분 충족)** — 제품이 폭넓게 쓸모는 있으나 **어느 세그먼트도 완벽히 충족 못 함**("넓고 얕음"). 한 세그먼트를 깊게 잡는 전략 필요.
- **미충족 우세(underserved):** `F-O-S-H` `F-O-A-H` `F-C-S-H` `F-C-A-H` `F-C-A-L` `R-O-A-H` `R-C-A-H` `R-C-A-L`
  → 두 갈래: ①고관여·분석형(확률·근거·투명성 갈증), ②불안·위기형(안전·정서지지·구조적 진단 부족)
- **우선 공략·대응(priority):** `R-C-A-H` `R-C-A-L`(안전 최우선) · `F-C-A-H`(관계유지 위기) · `F-O-S-H` `F-O-A-H`(지불의향 핵심)
