# 랜딩 — Functional Spec

| 항목 | 값 |
|------|-----|
| Route | `/` |
| File | `web/src/app/page.tsx` |
| Access | 공개 (미들웨어 보호 없음) |
| Rendering | Server (`export const dynamic = "force-dynamic"`, 세션 확인) |

## 목적
서비스 가치를 전달하고, 비로그인 사용자를 **무료 진단** 또는 **로그인**으로 보낸다.

## 동작
- 진입 시 세션 확인(`supabase.auth.getUser`, 서버 클라이언트 `@/lib/supabase/server`).
  - **로그인 상태 → `/diagnose`로 리다이렉트** (랜딩 노출 안 함).
  - 비로그인 → 랜딩 표시.

## UI 요소
- 상단 라벨: "고백해도 될까? 지금 연락해도 될까?"
- 헤드라인: "연애의 결정적 순간, **타이밍**을 분석해 드립니다"
- 서브카피: 내 상황과 상대 성향 데이터를 분석해 **언제·어떻게** 행동해야 성공 확률이 높은지 알려주는 AI 컨설턴트
- 1차 CTA: **"무료로 진단 시작 (로그인 없이)"** → `/diagnose`
- 2차 CTA: "로그인 · 회원가입" → `/login`
- 가치 포인트 3개:
  - 일반론 아닌 **내 상황** 맞춤 분석
  - '냉정한 거울' — 듣기 좋은 말 대신 솔직한 진단
  - 바로 쓸 수 있는 **연락 문구**까지

## 사용자 액션
| 액션 | 결과 |
|------|------|
| 무료로 진단 시작 (로그인 없이) | `/diagnose` (게스트 체험) |
| 로그인 · 회원가입 | `/login` |

## 상태
- 클라이언트 상태 없음(서버 컴포넌트). 서버에서 `user` 값만 계산해 리다이렉트 여부 결정.

## 데이터
- Reads: 현재 세션(로그인 여부, `supabase.auth.getUser`)
- Writes: 없음

## 엣지 케이스
- `getUser` 호출이 예외를 던지면 `try/catch`로 잡아 비로그인 취급 → 랜딩 정상 노출.
- 리다이렉트(`redirect`)는 내부적으로 throw를 사용하므로 `try` 블록 밖에서 호출.
