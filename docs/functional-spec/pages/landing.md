# 랜딩 — Functional Spec

| 항목 | 값 |
|------|-----|
| Route | `/` |
| File | `web/src/app/page.tsx` |
| Access | 공개 |
| Rendering | Server (dynamic, 세션 확인) |

## 목적
서비스 가치를 전달하고, 비로그인 사용자를 **무료 진단** 또는 **로그인**으로 보낸다.

## 동작
- 진입 시 세션 확인(`supabase.auth.getUser`).
  - **로그인 상태 → `/diagnose`로 리다이렉트** (랜딩 노출 안 함).
  - 비로그인 → 랜딩 표시.

## UI 요소
- 헤드라인: "연애의 결정적 순간, 타이밍을 분석해 드립니다"
- 서브카피: 언제·어떻게 행동할지 알려주는 AI 컨설턴트
- 1차 CTA: **"무료로 진단 시작 (로그인 없이)"** → `/diagnose`
- 2차 CTA: "로그인 · 회원가입" → `/login`
- 가치 포인트 3개 (맞춤 분석 / 냉정한 거울 / 연락 문구)

## 사용자 액션
| 액션 | 결과 |
|------|------|
| 무료로 진단 시작 | `/diagnose` (게스트 체험) |
| 로그인·회원가입 | `/login` |

## 데이터
- Reads: 현재 세션(로그인 여부) · Writes: 없음

## 엣지 케이스
- Supabase 미응답 시 `getUser`는 null 반환 → 랜딩 정상 노출.
